Resources for Data Authors

Data authors are classified as the individual scientist or data set creator that would like to publish their data set themselves. These have been traditionally thought of as the “one off” and “non-data manager” use cases. A subset of the tooling and data management practices are presented to this class of user so they are not overwhelmed. Of course though, if they would like to learn about all the bells and whistles, we encourage them to check out resources for data managers.

Now ... where are you going to find the time to publish data in all this?

It turns out the data publication process maps nicely onto the research life cycle and really only entails a little better bookkeeping in the data collection, organization, and cleaning phases, and then upload of those metadata and data to a repository.

Note: Our data publication submission interface (ezEML) Allows you to describe your data throughout the collection, organization, and cleaning phases of the research life cycle, thereby greatly simplifying the amount of work required during the upload step. This bookkeeping is not only beneficial to the publication face but also serves as an authoritative set of descriptions you can share with others.

The Data Package

The data package is the unit of publication. It’s an assemblage of science metadata and one or more science data objects (these are the parts that you are responsible for) and include a quality report that's generated at the time of upload and describes how well your metadata match the actual contents of your data objects. These three objects are then described by what we call a resource map which is essentially a manifest of all these items. All this content is then bundled up stamped with a DOI and put into long-term storage for reuse.

Data packages have some important features. First, they're immutable which makes them suitable for reproducible research. Second, data packages are versionable thereby accommodating updates and improvements through time. And finally, data packages are citable through globally unique and persistent digital object identifiers.

What to Include in a Data Package?

If you will be publishing data in support of a manuscript, then include data required to reproduce the results. Consider publishing the minimally processed version of the data, so they can be reused in other contexts with scripting the processing and analysis in your computational language of choice so your results are reproducible. Be sure to include links to any source data used in your work (See the concept of provenance metadata). [Figure: organize_reproducible.svg]

If you are publishing data not as part of a manuscript but as part of an ongoing time series serving context and general data reuse, then see our data management resource on “organizing data into publishable units”.
